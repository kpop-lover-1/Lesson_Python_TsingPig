先按照h的降序排列，在相同h的内部就按照k的升序排列
## 🌟 题目回顾

> 给定一个数组 `people`，其中每个元素是 `[h_i, k_i]`：
> 
> - `h_i` 是第 i 个人的身高；
> - `k_i` 表示在他前面**正好有 k_i 个**身高 **大于等于** 他的人。
> 
> 要求：重新构造出正确的队列顺序。

### 示例：
```
输入: people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
输出: [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
```

---

## ✅ 思路核心（贪心 + 排序）

我们要解决的问题是：  
**如何安排每个人的站位，使得每个人前面恰好有 k_i 个比他高或一样高的人？**
### 🔍 关键观察：
- 如果一个人很高，那么他在队伍中会“挡住”后面矮的人。
- 所以，**先处理高的人**，再处理矮的人，可以避免干扰。
👉 这就是 **贪心思想** 的体现！
---

## 🚀 代码逐行解析
```
def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
    # 第一步：排序
    people.sort(key=lambda x: (-x[0], x[1]))
    
    # 第二步：构建结果队列
    res = []
    for i, p in enumerate(people):
        h, k = p[0], p[1]
        if k == i:
            res.append(p)
        elif k < i:
            res.insert(k, p)
    return res
```

---

### 🔹 第一步：排序 `people.sort(key=lambda x: (-x[0], x[1]))`

#### 1. `lambda x: (-x[0], x[1])` 是什么意思？

- `x[0]` 是身高 `h`
- `x[1]` 是前面人数 `k`
- `-x[0]`：负号表示**按身高从大到小排**
- `( -x[0], x[1] )`：是一个元组，Python 按照元组的字典序排序
📌 **排序规则是：**
1. 先按 **身高降序**（高的在前）
2. 身高相同时，按 **k 值升序**（即前面人数少的在前）
#### 举个例子：
原始数据：
```
[[7,0], [4,4], [7,1], [5,0], [6,1], [5,2]]
```
排序后变成：
```
[[7,0], [7,1], [6,1], [5,0], [5,2], [4,4]]
```

- 身高 7 的两个：`[7,0]` 和 `[7,1]` → 因为 `0 < 1`，所以 `[7,0]` 在前
- 然后是 6 → `[6,1]`
- 然后是 5 → `[5,0], [5,2]`
- 最后是 4 → `[4,4]`

✅ 排序完成！

---

### 🔹 第二步：遍历并插入
```
res = []  # 结果队列，初始为空
for i, p in enumerate(people):  # i 是索引，p 是当前人 [h,k]
    h, k = p[0], p[1]
    if k == i:
        res.append(p)
    elif k < i:
        res.insert(k, p)
```
我们现在要**一个一个把人放进结果队列 `res` 中**。
但注意：**我们是从高到低处理的**，所以高的已经先处理了。
```
for i, p in enumerate(people):
```
- `enumerate(people)` 会返回 `(索引, 元素)` 对
- `i` 是当前元素在排序后列表中的位置（从 0 开始）
- `p` 是当前的人，比如 `[7,0]`
---

#### ✅ 第四轮：i=3, p=[5,0]
- `h=5`, `k=0`
- `k == i`？→ `0 == 3` ❌
- `k < i`？→ `0 < 3` ✔️
- 所以：`res.insert(0, [5,0])`
- 插入到索引 0

res = [[5,0], [7,0], [6,1], [7,1]]

现在 `[5,0]` 在最前面，它前面没有人，k=0，满足 ✅

- `k == i`？→ `2 == 4` ❌

## 🎯 为什么要这样排序？
### ✅ 为什么先按身高降序？
- 因为高的会挡住矮的，所以**先安排高的**，就不会影响矮的。
- 一旦矮的放进去，后面的高个子不会改变它前面的人数。
### ✅ 为什么身高相同时按 k 升序？
- 比如两个都是身高 7，一个是 `[7,0]`，一个是 `[7,1]`
- `[7,0]` 要求前面没人，必须放在最前面
- 所以它应该先被处理，否则后面的人插进来会影响它

## 💡 类比：插入排序（你贴的图）
你的第三张图是 **插入排序**，其实这里的逻辑很像：
- 我们维护一个已排好序的队列 `res`
- 每次拿一个人进来，**插入到正确的位置**
- 插入时，用 `insert(k, person)` 实现
这正是 **插入排序的思想**！
只不过这里不是比较数值大小，而是根据“前面有多少个 ≥ 我”的条件来决定插入位置。

---

## ✅ 总结：算法步骤

|步骤|内容|
|---|---|
|1|对所有人按 **身高降序**，**k 值升序** 排序|
|2|依次遍历每个人|
|3|如果当前人的 `k == 当前索引`，直接追加到末尾|
|4|如果 `k < 当前索引`，就插入到索引 `k` 处|

---
## 🧠 核心思想一句话总结：
> **先处理高个子，再处理矮个子；高个子不会影响矮个子的相对位置，因此我们可以安全地插入。**


这一行是 Python 中一个方法的函数签名，通常出现在类的定义中。我们来逐部分解释它的含义：
```
def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
```

### 1. `def reconstructQueue(...)`
- 这是一个函数（或方法）的定义。
- 函数名是 `reconstructQueue`，表示这个方法的作用可能是“重建队列”。
### 2. `self`
- `self` 是 Python 类方法中的第一个参数，代表调用该方法的对象实例。
- 它是面向对象编程中用来访问当前对象属性和方法的惯例。
### 3. `people: List[List[int]]`
- `people` 是这个方法的输入参数。
- 类型注解 `List[List[int]]` 表示 `people` 是一个**列表的列表**，每个内部列表包含整数。
- 例如：`[[7,0], [4,4], [7,1], [5,0], [6,1], [5,2]]`
    - 每个子列表 `[h, k]` 可能代表一个人的身高 `h` 和前面有 `k` 个人比他高。
### 4. `-> List[List[int]]`
- 这是返回值的类型注解。
- 表示该方法会返回一个与输入格式相同的结构：`List[List[int]]`，即一个二维整数列表。
---
### 综合解释：
这行代码定义了一个名为 `reconstructQueue` 的方法，它接收一个二维整数列表 `people`（比如每个人的身高和前面比他高的人数），并返回一个重新排列后的二维列表，表示按照某种规则重建后的队列顺序。
```
def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
```
意思是：
> 定义一个方法 `reconstructQueue`，它接受一个由 `[身高, 前面更高人数]` 组成的二维列表作为输入，返回一个重新排序后的二维列表，表示符合要求的队列。


你提到的这种语法是 **Python 的类型注解（Type Hints）**，它是从 Python 3.5 开始引入的一项功能（通过 [PEP 484](https://peps.python.org/pep-0484/)），目的是让代码更清晰、更容易被工具（如 IDE、类型检查器）理解和检查。

我们以这行代码为例：
```
def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
```

---
## ✅ 1. 基本函数定义（无类型注解）
先看没有类型注解的普通写法：
```
def add(a, b):
    return a + b
```
Python 是动态语言，`a` 和 `b` 可以是任何类型。但这样别人读代码时不知道你期望传入什么类型。

---
## ✅ 2. 加上类型注解：参数和返回值
```
def add(a: int, b: int) -> int:
    return a + b
```
- `a: int` 表示参数 `a` 应该是一个整数。
- `-> int` 表示这个函数返回一个整数。
- 这些**不会在运行时强制检查类型**，只是“提示”（所以叫 _hint_），但像 PyCharm、mypy、VS Code 等工具会用它来帮你发现错误。

---
## ✅ 3. 复杂类型：List、Dict 等
对于列表、字典等容器类型，不能直接写 `list`，而要用 **泛型类型**，来自 `typing` 模块。
### 导入方式（Python < 3.9）：
```
from typing import List, Dict, Tuple
```
### 示例：
```
def get_names(students: List[str]) -> List[str]:
    return [name.upper() for name in students]
```
- `List[str]`：表示这是一个字符串列表。

---
## ✅ 4. 嵌套类型：List[List[int]]

你的例子中：
```
people: List[List[int]]
```

意思是：
- `people` 是一个 **列表**
- 它的每个元素也是一个 **列表**
- 而这些内部列表的元素都是 **整数**
例如：
```
[[7, 0], [5, 2], [6, 1]]  # ✅ 符合 List[List[int]]
```

---
## ✅ 5. 返回值类型注解
```
-> List[List[int]]
```
表示函数返回的值也是这种二维整数列表。

---
## ✅ 6. self 参数
在类的方法中，第一个参数总是 `self`，代表当前对象实例：
```
class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        # self 指向当前 Solution 实例
        pass
```

`self` 通常不加类型注解（虽然也可以加，比如 `self: "Solution"`），一般省略。

---
## ✅ 7. Python 3.9+ 的简化写法
从 Python 3.9 开始，你可以直接用内置的 `list`、`dict` 等作为泛型，不需要从 `typing` 导入：
```
def f(people: list[list[int]]) -> list[list[int]]:
    ...
```
但如果你要兼容旧版本（比如 LeetCode 环境可能是 3.8），还是建议用 `List`。

---
## ✅ 8. 实际完整例子
```
from typing import List

class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        # 按身高降序，k 升序排序
        people.sort(key=lambda x: (-x[0], x[1]))
        queue = []
        for p in people:
            queue.insert(p[1], p)
        return queue
```
