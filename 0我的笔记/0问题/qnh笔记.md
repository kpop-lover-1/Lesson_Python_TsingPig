# IEEE 754 浮点数表示（C语言）

## 基本结构（以 `float` 和 `double` 为例）

| 类型       | 总位数    | 符号位   | 阶码（指数） | 尾数（有效数字） | 精度     |
| -------- | ------ | ----- | ------ | -------- | ------ |
| `float`  | 32 bit | 1 bit | 8 bit  | 23 bit   | 单精度浮点数 |
| `double` | 64 bit | 1 bit | 11 bit | 52 bit   | 双精度浮点数 |

> 📝 注：小数点前的 `1` 在规格化形式中**隐含不存储**（即“隐藏位”），仅在非规格化数中为 0。

---

## 阶码的存储方式

- **使用移码（biased exponent）表示**
  - `float`：偏置值 = $2^{8-1} - 1 = 127$
    - 存储值 = 实际指数 + 127  
    - 例：指数为 `-126` → 存储为 $-126 + 127 = 1$ → 二进制 `00000001`
  - `double`：偏置值 = $2^{11-1} - 1 = 1023$

> ⚠️ 规格化浮点数的阶码 **不全为 0，也不全为 1**  
> （全 0 表示非规格化数/零/次正规数；全 1 表示无穷大或 NaN）

---

## 尾数（Mantissa / Fraction）取值范围

- 对于规格化数，尾数隐含前导 `1.`，即实际有效数字为 `1.f`（f 为存储的尾数部分）
- 尾数真值范围：
  - 最小：$1.0 \times 2^{-126}$（对应阶码=1，尾数全0）
  - 最大：$(2 - 2^{-23}) \times 2^{127}$（float）  
    或 $(2 - 2^{-52}) \times 2^{1023}$（double）

> ✅ 推导示例（float）：
> $$
(1 - 2^{-23}) \times 2^{126} = 2^{126} - 2^{126-23} = 2^{126} - 2^{103}
$$
> 但更标准写法是：
> $$
\text{最大值} = (2 - 2^{-23}) \times 2^{127} = 2^{128} - 2^{104}
$$

---

## 非规格化数（Denormalized / Subnormal Numbers）

- 当阶码全为 0（即存储值=0），且尾数 ≠ 0 时，为非规格化数
- 此时隐含前导位为 `0.` 而非 `1.`
- 数值公式：  
  $$
  (-1)^s \times 0.f \times 2^{-126} \quad (\text{float})
  $$
- 举例：`0 00000000 000...001`  
  → 符号=0，阶码=0 ⇒ 指数 = $-126$  
  → 尾数 = $1 \times 2^{-23}$  
  → 值 = $1.0 \times 2^{-23} \times 2^{-126} = 2^{-149}$

> 💡 非规格化数用于表示**极小值**，避免下溢（underflow）直接归零，实现“渐进下溢”。

---

## 特殊值

| 阶码 | 尾数 | 含义               |
|------|------|--------------------|
| 全 1 | 全 0 | ±∞（无穷大）       |
| 全 1 | 非 0 | NaN（非数）        |
| 全 0 | 全 0 | ±0（正零/负零）    |
| 全 0 | 非 0 | 非规格化数         |

> 📌 若下溢至机器零，可设置浮点数下溢异常标志位（如 IEEE 754 的 *underflow flag*）。

# 通信方式对比

## 1. 独立请求方式（Independent Request）

### 优点：
1. 响应速度快：信号可直接从控制器发送到相关设备；
2. 对优先级高的控制相当灵活。

### 缺点：
1. 控制线数量多  
   - 若有 $n$ 个设备，则需 $2n+1$ 条控制线  
     （其中 $n$ 条为 BS 线 — Bus Select？或 Busy Signal？常见为 *Bus Grant* 或 *Device Select*）
2. 总线的控制逻辑更加复杂。

---

## 2. 分布式仲裁方式

### 总线定时分类：
| 类型             | 描述                                                                 |
|------------------|----------------------------------------------------------------------|
| 同步通信         | 同步定时方式：由统一时钟控制数据传送（所有部件同步于同一时钟）     |
| 异步通信         | 异步定时方式：采用应答方式（如 `REQ`/`ACK`），无公共时钟标准       |
| 半同步通信       | 同步 + 异步结合：主设备发时钟，从设备用应答信号微调 timing         |

> 📌 同步通信适用于总线长度较短、各部件时序较接近的系统。

#### 异步通信
- **优点**：
  - 总线周期长度可变，能适应工作速度差异较大的部件；
  - 部件间可独立进行信息交换，自动适应时间配合。
- **缺点**：速度比同步定时方式慢。

#### 半同步通信
> 在统一时钟基础上，增加一个“等待”响应信号（如 `WAIT`），用于延长周期。

#### 分离式通信（Split Transaction）
- 各模块均有权申请总线；
- **子周期1**：主模块申请并占用总线，使用完毕后释放总线使用权；
- **子周期2**：从模块申请占用总线，将各种信息送至总线上。

> ✅ 优势：提高总线利用率，支持并发操作。
